## Root cause: why every session diverged

The divergences are not random. Decisions made mid-project (no Tailwind, no Rubber
naming, all content via Notion) were never written into the docs, and the migrations
they triggered were never driven to completion. So:

- **CLAUDE.md** (the file every session reads first) is frozen at 2026-03-23.
  17 of ~22 of its current-state claims are false: it says Tailwind v4 stack, mock
  data, 19 pages, `/tires/` routes, "create Notion DBs next". Reality: 1050 pages,
  351 products synced from Notion on 2026-06-10, `/products/` routes.
- The **real architecture is documented nowhere** — the only accurate description
  of the data pipeline on disk is a code comment in `src/lib/data.ts:1-9`.
- The newest doc (**wp-page-parity.md**, 2026-06-09, otherwise ~90% accurate)
  prescribes 25+ new components in the banned Rubber*/rubbermorphism vocabulary.
- **Two design-token systems are documented** (WP palette, Helvetica Neue, 1170px
  in CLAUDE.md/IMPLEMENTATION-PLAN) and a **third undocumented one shipped**
  ("Rubbermorphism" tokens in global.css). None reconciled.

Every new session read stale docs, pattern-matched to whichever era it landed in,
and manufactured more divergence. The fix is procedural as much as technical
(see Path Forward, Phase 1).

## The five rot clusters (verified findings, condensed)

### 1. Three half-finished migrations running simultaneously

**Tailwind → central CSS (~50% done; currently load-bearing both ways)**
- `global.css:1` is `@import 'tailwindcss'`; the `@theme` block holds ~50 design
  tokens — removing Tailwind today breaks every page (tokens die with it).
- 1,858+ utility occurrences across 26 of 49 `.astro` files. Fully-Tailwind
  surfaces: `NotionBlocks.astro` (all article/product body rendering),
  `BaseLayout` body, `RubberHeader` chrome, `Footer`, resources index/detail,
  contact, design gallery.
- `prose-content` class is referenced on detail pages but **defined nowhere**.
- RTL depends on `rtl:scale-x-[-1]` Tailwind utilities in 12 files — removing
  Tailwind without porting these silently breaks Arabic.
- 2 scoped `<style>` blocks remain (contact page, Specimen).

**Rubber → semantic naming (barely started)**
- 17 `Rubber*` component files; 104 unique `.rubber-*` selectors across 18 of 26
  component stylesheets; 117 rubber-class references in 19 components; literal
  `'rubber-chip'` strings in ProductsViewer client JS.
- Both conventions coexist in one directory (`industry-card.css` vs `chip.css`).

**Old-gen → new-gen components (usage migrated, deletion never done)**
- Hard-dead: `Header.astro`, `MegaNav.astro` (no importers).
- 7 superseded duplicates (ProductCard, ArticleCard, FeaturedCard, IndustryCard,
  IndustryBadge, Hero, SpecTable) kept alive **only** by the design gallery.
- 699 lines of dead mock data (`mock-tires.ts`, `mock-articles.ts`).
- Empty dirs: `src/components/tire/`, `src/pages/{en,ar-ae,zh-hans}/`.

### 2. Locale slug identity is broken (worst user-facing defect)

- `products/[industry]/index.astro` builds routes from **EN slugs for all locales**
  while every nav link uses the **locale-specific slug** from Notion
  (`tbr-ar-ae`, `rubber-tracks-zh-hant`, `agricultural-tractor-tires`…).
  Result: **5 industry-hub URLs linked from the main nav 404** (2 ar-ae, 3 zh-hant),
  the EN-slug routes that do build are empty redirect stubs, and ~36 product
  breadcrumbs point at 404s.
- `LanguageSwitcher` does a blind locale-prefix swap: 404s on 11/117 products with
  divergent slugs and all 51 EN-only articles. `trid`/`translationIds` are synced
  into every snapshot **and consumed by nothing**. Zero hreflang alternates.
- Homepage Rubber Tracks dedup filters by EN slug only → ar-ae/zh-hant homepages
  show a duplicated tile EN doesn't have (structure parity violation).
- Simplified Chinese inside the Traditional locale: `zh-hant.json:103-105`
  (contact.*) + contact page region names (`欧盟`, `亚洲、中东和非洲`).
- ~20 aria-labels hardcoded in English across all locales.

### 3. Content pipeline open loops (content not maintainable via Notion)

- **Product specs**: generated from the **frozen May-4 WP export**
  (`build-product-specs.ts` ← `scripts/output/tires-*.json`). Editing specs in
  Notion changes nothing; products added in Notion can never get structured specs.
  The Notion copies are too lossy to parse (import collapsed newlines), so closing
  the loop needs a structured Specs DB seeded from the normalized JSON
  (same playbook as Pages/Promos, done 2026-06-10).
- **Contact offices**: full editorial dataset + inline tri-locale translation map
  hardcoded in page frontmatter (`contact/index.astro:23-61`).
- **Featured products**: hardcoded slug list `['agrixtra-85','ms453','ms600','ms709']`
  in `index.astro`; Notion has no Featured flag.
- **pages.json covers only `home`** — products/resources/contact have no Pages
  record; `/products` hero image is a literal style attribute.
- **Images**: only 5 files in `public/images` (referenced from Notion values AND
  duplicated as hardcoded fallbacks); **~900 product/article images hotlink the
  legacy maxamtire.com** — a launch-blocking external dependency.
- **Events**: synced, block-split, filterable in nav — but no route ever renders
  them; the 'event' filter can never match; all 6 events EN-only.
- **Dead-end CTAs**: `/testimonials` and `/sustainability` linked from the homepage
  in all 3 locales; neither route exists.
- Sentence-length marketing copy lives in translations JSON (`home.resources_lead`).
- 50 of 121 translation keys are orphans (38 `category.*` duplicating Notion
  taxonomy names + 12 literals).

### 4. Repo/ops fragility (bus factor = 1)

- **`.gitignore:35-36` ignores ALL of `scripts/`** — the live sync pipeline
  (`sync-from-notion.ts`, `build-product-specs.ts`) and `notion-ids.json`
  (required by every sync) exist on one machine only. `git ls-files scripts/` = 0.
- **5 snapshot files statically imported by `data.ts` are untracked**
  (pages, page-promos, product-specs ×3) — a fresh clone **fails `astro build`**.
- Last commit 2026-05-04; 5 weeks of drift uncommitted.
- Latest sync ran `--without-blocks`: row metadata is 2026-06-10, block bodies are
  2026-05-04 — two eras served simultaneously; missing sidecars fail silently to
  empty page bodies.
- `import-pages-to-notion.ts` is non-idempotent (re-run duplicates rows) — by
  design, but undocumented outside its header.
- 13 probe/debug scripts + 15MB `scripts/output/` with only 4 load-bearing files.

### 5. Layout system fragmentation

- **5 parallel container/gutter systems** (Section container; products
  `--page-gutter`; `.tire-page` ad hoc clone; Tailwind `max-w-[…] px-4` on
  resources/contact/Footer — fixed 1rem gutter that doesn't match; `max-w-3xl`
  reading column). Content edges don't align across pages.
- 6 h1 treatments, 5 hero systems (+1 dead), 4 section-spacing systems,
  4+ grid implementations.
- `.products-hero` duplicates Section's bg/overlay machinery; form atom CSS
  triplicated (~90% identical input/select/textarea); chamfer `::before` block

## What is solid (keep, do not touch)

- **The pipeline**: Notion → `sync-from-notion.ts` → snapshots → `data.ts` →
  build. Offline, fast (1050 pages / ~13s), zero runtime Notion leakage,
  Pages+Promos loop closed 2026-06-10 with hard-fail guards.
- **`src/lib/data.ts`**: 19/25 exports in active use, consistent accessor
  patterns, blocks lazily fs-read (31MB stays out of the bundle).
- **The new-generation components + 29 central stylesheets** (0 orphans, all
  imported by global.css): Section/Grid/chips/buttons/cards/spec-viewer/
  gallery/filter-sidebar/ProductsViewer. The pattern is proven — home, products,
  industry hub, tire detail already live on it.
- **i18n foundations**: 100% key parity across the 3 locale files; `t()`/
  `localePath()`/`getDir()`; NotionBlocks covers 11/11 observed block types.
- Git history (back to scaffold) — salvage and bisection are possible.

## Path forward (sequenced; each phase ends with the old path deleted)

**Phase 0 — Stop the bleeding (repo integrity, ~half session)**
Fix `.gitignore` (`scripts/output/` only), commit the pipeline scripts +
notion-ids.json + all 5 untracked snapshots + current drift. Run full
`npm run sync` (with blocks). Delete: Header.astro, MegaNav.astro, mocks,
empty dirs. Decide design-gallery fate (it's the only thing keeping 7 dead
components alive).

**Phase 1 — Docs become the contract (~half session) ← highest leverage**
Rewrite CLAUDE.md from scratch: real architecture (sync → snapshots → data.ts),
the standing decisions (no Tailwind, no Rubber, Notion-only content, container/
gutter rules, locale parity), current state, known open loops. Archive
IMPLEMENTATION-PLAN.md + DESIGN-SYSTEM.md as `docs/archive/`. De-Rubber
wp-page-parity.md. This is what stops session drift.

**Phase 2 — Locale slug identity (~1 session)**
Normalize industry/product slugs to locale-stable at sync time (trid-keyed),
regenerate routes per-locale, make LanguageSwitcher + breadcrumbs resolve via
the trid map, emit hreflang. Fix Simplified→Traditional strings. Add a
build-time internal-link assertion (any nav href to a non-generated route
fails the build).

**Phase 3 — Finish the Tailwind removal (~1-2 sessions)**
Move `@theme` tokens to `:root`. Write `prose-content.css` for NotionBlocks.
Port RTL `rtl:` utilities to `[dir='rtl']` rules. Convert the 3 Tailwind pages
(resources ×2, contact) + Footer/BaseLayout/header chrome to central CSS.
Then delete `@import 'tailwindcss'`, the vite plugin, and the packages —
making regression impossible.

**Phase 4 — Rename pass (~1 session, mechanical)**
RubberX.astro → X.astro (17 files), `.rubber-*` → semantic (18 stylesheets,
19 components, 2 client scripts). Grep-assert zero `rubber` afterward.
Unify the 5 container systems to the sanctioned 2; collapse form-atom CSS.

**Phase 5 — Close the content loops (~1-2 sessions)**
Product Specs Notion DB seeded from the normalized JSON (same playbook as
Pages/Promos); Offices DB; Featured flag; Pages records for products/resources/
contact; re-host the ~900 hotlinked images (launch blocker); events → build the
route or remove from nav; build or unlink /testimonials + /sustainability;
purge orphan translation keys.
