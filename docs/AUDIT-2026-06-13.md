## The verdict in one paragraph

The multi-language port was done **out of sequence and made things worse on net.**
Real wins exist — the locale-slug 404 cluster is genuinely fixed (canonical
English `urlSlug` keyed by trid; build green at 4467 pages) and the ContentLanguage/
Locale/alias split is well-implemented. But the prior audit's **two prerequisite
phases were skipped entirely** (Phase 0 repo integrity, Phase 1 docs contract),
and the language work piled ~470 uncommitted files onto an already-unbuildable
repo, shipped 6 empty locales, and created ~918 dead duplicate files. This is the
exact failure mode the 2026-06-10 audit named: adding breadth before securing the
foundation.

## CRITICAL — do these before anything else (repo integrity, unchanged from prior audit + worse)

1. **Fresh clone cannot build.** `data.ts` and `i18n.ts` statically import 14
   files (7 `product-specs.<lang>.json` + 7 `translations/<lang>.json`) that are
   **untracked** (not ignored — never `git add`ed). Was 5 files at the prior
   audit; now 14. The translations are hand-authored and NOT regenerable.
2. **`.gitignore:33` ignores all of `scripts/`** → the entire sync pipeline
   (129 files, 36MB) and `notion-ids.json` (sync's identity map, required every
   run) exist on this machine only. `git ls-files scripts` = 0. Also ignores
   `CLAUDE.md` and `docs/` — the contract and audit can't be versioned.
3. **Zero commits since 2026-06-10.** 470 files of drift (437 untracked + 33
   modified). Every bit of the locale split, slug fix, resources model, and
   7-language work is in the working tree only. **Bus factor 1; machine death =
   total loss of 3+ days of irreplaceable work.**
4. **6 locales ship empty.** es/fr/it/ja/pt-pt/ru (+ aliases es-mx/fr-ca) are
   routable with translated UI + spec files, but `products.json`/`articles.json`
   were never synced — `/es/products/` renders **zero products** (1 page vs en's
   123). Blank homepages too (Pages snapshot has 3 langs, Notion has 10).

## MAJOR — the waste the user caught (newly introduced by the language port)

5. **~918 dead article block files (8.6MB).** `article-{de,ar-ae,zh-hant}-*`
   sidecars are **byte-identical English** (only Notion ids/timestamps differ).
   `data.ts` `RESOURCE_LANG='en'` means **no code path ever reads them.** Syncing
